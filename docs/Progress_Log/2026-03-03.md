# 2026-03-03 (화): 실사용 에셋 선별 LFS 추적 적용

#### **실사용 에셋만 Git LFS로 추적하도록 저장소 정책 전환**

* **오늘 반영한 작업**
* `gitignore`로 제외된 서드파티 에셋 루트(`CombatGirlsCharacterPack`, `FourEvilDragonsPBR`, `UNI VFX`)를 대상으로 GUID 기반 의존성 폐쇄를 계산해 실사용/미사용을 분류했다.
* `Assets/TextMesh Pro` 루트는 사용자 요청대로 커밋 대상에서 제외했다.
* 실사용 에셋만 강제 스테이징(`git add -f`)해 `.meta` 및 상위 폴더 `.meta`까지 함께 포함했다.
* `.gitattributes`에 실사용 경로 중심 LFS 규칙을 추가했다.
  * `Assets/CombatGirlsCharacterPack/**/*.fbx`
  * `Assets/FourEvilDragonsPBR/**/*.fbx`
  * `Assets/FourEvilDragonsPBR/Texture/DragonUsurper/Red/*.png`
  * `Assets/UNI VFX/**/*.fbx`
* 최종 스테이징 집계:
  * 총 122파일 / 70.59MB
  * LFS 포인터 대상 35파일(`.fbx` 30, `.png` 5)

* **맥락노트**
* 다른 PC에서 `git pull` 후 즉시 실행이 깨지는 원인은 런타임 필수 에셋이 `.gitignore`로 누락된 상태였기 때문이다.
* 전체 서드파티 팩을 통째로 올리면 무료 LFS 한도 소모가 커지므로, 현재 프로젝트가 실제 참조하는 집합만 선별해 추적하는 방식이 필요했다.
* Unity 에셋 참조 안정성을 위해 원본 파일과 `.meta`를 쌍으로 관리하고, 중간 폴더 `.meta` 누락도 함께 방지했다.

* **기술적 고려**
* **무엇을 발견했는가**
* 후보 에셋 루트 전체는 642.6MB였지만, 실제 참조 폐쇄 집합은 70.59MB로 크게 작았다.
* 미사용 바이너리(`.fbx`, `.png`)가 522.77MB로 다수였고, 특히 드래곤 색상 변형 텍스처가 용량을 크게 차지했다.
* **무엇을 수정했는가**
* 실사용 세트만 Git/LFS 추적 대상으로 선별해 스테이징했고, `Assets/TextMesh Pro` 루트는 제외했다.
* LFS 규칙을 확장자 전체가 아닌 실사용 경로 중심으로 제한해 불필요한 대역폭/스토리지 사용을 줄였다.
* **왜 그렇게 판단했는가**
* 목표는 "다른 PC에서 pull 후 즉시 실행 가능"과 "무료 LFS 한도 유지"를 동시에 만족하는 것이다.
* GUID 의존성 폐쇄 기반 선별은 수동 추정보다 누락 위험이 낮고, 실제 런타임 요구와 저장소 용량의 균형을 맞추기에 가장 실용적이다.
